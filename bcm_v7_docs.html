02Koncepcja dual-display

Zasada: dwa ekrany, dwie funkcje, zero przełączania

Zamiast jednego ekranu z menu i przełączaniem trybów — dwa niezależne ekrany pracujące równolegle. Kierowca widzi wszystko jednocześnie bez konieczności przełączania między BCM a multimediami.

🖥️ Ekran 4.3" — BCM (zarządzanie autem)

Lokalizacja: W desce rozdzielczej (miejsce po zegarku / wskaźniku lub custom slot)

HDMI-1 z Orange Pi 5 Plus

Stała zawartość:

• Dashboard: temp silnika, spalanie, paliwo, RPM, prędkość

• Temperatura zewnętrzna + ostrzeżenia oblodzenia

• Trip computer + serwis

• Kamera cofania + czujniki parkowania (override przy wstecznym)

Sterowanie: Kontroler obrotowy na tunelu

🖥️ Ekran 7" — Multimedia

Lokalizacja: Miejsce po radiu (1DIN + kieszeń, ramka 3D)

HDMI-2 z Orange Pi 5 Plus

Stała zawartość:

• Android Auto / CarPlay (pełnoekranowy)

• Muzyka z telefonu (BT A2DP) z wizualizacją

• Radio FM (opcja, RTL-SDR)

• Podgląd dashcam (opcjonalny)

Sterowanie: Touch 7" + pilot BT na kierownicy

Dlaczego Orange Pi 5 Plus

Dual HDMI wymaga SBC z dwoma wyjściami HDMI. Orange Pi 5 Plus ma 2× full-size HDMI 2.1, wbudowane WiFi 6 + BT 5.0, i pełny 40-pinowy GPIO header (jak RPi).

DECYZJA: Orange Pi 5 Plus (4GB). Kluczowe: 2× HDMI (dual-display bez adapterów), pełny 40-pin GPIO (prostsza integracja), 2× USB 3.0 (grabber + pendrive na dedykowanych portach). WiFi 6 zamiast 6E — różnica nieistotna dla wireless AA. BT 5.0 zamiast 5.3 — A2DP Sink i BLE działają identycznie.

Konfiguracja dual HDMI

HDMI-1 (port bliżej USB): → Ekran 4.3" BCM (800×480) HDMI-2 (port bliżej GPIO): → Ekran 7" multimedia (1024×600) Armbian/OPi OS konfiguracja: /dev/fb0 → HDMI-1 (BCM renderer: Pillow / PyGame) /dev/fb1 → HDMI-2 (OpenAuto Pro: EGL / SDL2) Xrandr: xrandr --output HDMI-1 --mode 800x480 --pos 0x0 xrandr --output HDMI-2 --mode 1024x600 --pos 800x0 (extended desktop, nie mirror) Lub: Wayland z dwoma niezależnymi kompozytorami (wlroots).

06Sterowanie — kontroler + pilot BT na kierownicy ZMIANA

Dwa urządzenia sterujące

🎛️ Kontroler obrotowy (tunel środkowy)

Steruje: Ekran 4.3" (BCM)

Połączenie: USB HID (Arduino Pro Micro)

Funkcje:

• Enkoder obrót: nawigacja ekranów BCM

• Enkoder push: akcja kontekstowa (reset trip, serwis)

• Długi push: zmiana grupy A→B→C

• Opcjonalnie: przełączanie trybów na 7"

🎮 Pilot BT na kierownicy

Steruje: Audio + ekran 7" (multimedia)

Połączenie: Bluetooth HID (BLE)

Bateria: CR2032 (~1-2 lata)

Funkcje:

• 🔊 VOL+ / VOL- (głośność PipeWire → DAC)

• ⏭ NEXT / ⏮ PREV (następny/poprzedni utwór)

• ▶⏸ PLAY/PAUSE

• 📞 PHONE (odbierz/odrzuć połączenie)

REKOMENDACJA: Gotowy pilot BT. Nie trzeba budować — gotowe produkty za 25-45 PLN działają jako standardowy BT HID Keyboard/Consumer Control. Linux (BlueZ) paruje je automatycznie. Klawisze mapowane na HID Consumer keycodes (VOLUME_UP, VOLUME_DOWN, MEDIA_NEXT, MEDIA_PREV, PLAY_PAUSE, PHONE_ANSWER). Małe, ładne, CR2032 trzyma rok+.

Opcja B: Custom ESP32-C3 pilot (zaawansowana)

Jeśli gotowy pilot nie spełnia wymagań — własny na bazie ESP32-C3 (BLE HID native, ultra-low power, CR2032). Koszt: ~25 PLN (moduł) + przyciski + obudowa 3D. Firmware: Arduino BLE HID library. Pełna kontrola nad mapowaniem klawiszy.

08System audio 4.1 — DAC + AMP + sub

Bez zmian w architekturze od v6: USB DAC (PCM5102A / ES9038Q2M) → kabel RCA → wzmacniacz 4.1 Class-D TPA3116D2 (4×50W + 100W sub) → fabryczne głośniki AR156 + subwoofer 8" w bagażniku. Radio usunięte, BCM = pełny head-unit.

Mikrofon hands-free

USB mini mikrofon (klips na podsufitce/lusterku, ~20 PLN). PipeWire routing: mic → AA voice input / BT HFP. Auto-ducking muzyki przy rozmowie.

11Czujnik temperatury DS18B20 + oblodzenie

Bez zmian: DS18B20 (1-Wire), ±0.5°C, pod zderzakiem. ≤3°C + spadek → overlay na 4.3" (3s) + bip przez głośniki. ≤0°C → ❄ czerwona stale na pasku. Cooldown 10 min.

12Trip Computer + serwis

Bez zmian: Trip A (ręczny reset), dane z K-Line. Serwis: olej 15000km/12mies., opony sez., przegląd. Potwierdzenie enkodera push na ekranie C2 (4.3").

13Dashcam przód + tył

Bez zmian: 2× AHD 720P → 4-ch USB grabber → GStreamer pipeline → HW H.264 → pendrive 128GB. Loop recording ~47h. Kamera tył = kamera cofania (dual role). Opcjonalny podgląd na ekranie 7" (kafelek dashcam).

18Schematy elektryczne

18.1 L9637D → UART (K-Line)

+12V → [510Ω pull-up] → K-Line (OBD pin 7) → L9637D pin 1 L9637D pin 3 (RX) → OPi5+ UART RXD | pin 4 (TX) ← OPi5+ UART TXD L9637D pin 5 (VS) ← +12V | pin 2 (GND) → GND | [100nF] VS-GND

18.2 Dual backlight MOSFET (2× niezależne)

Ekran 4.3" backlight: GPIO_PWM_A → [1kΩ] → BC547 → IRLZ44N-A → Display 4.3" GND (przerwana) Ekran 7" backlight: GPIO_PWM_B → [1kΩ] → BC547 → IRLZ44N-B → Display 7" GND (przerwana) Niezależne sterowanie: 4.3" OFF (standby) + 7" OFF (standby) 4.3" ON + 7" ON (normal) Indywidualne fade-in/fade-out

18.3 System audio 4.1

OPi 5 Plus USB → [DAC USB PCM5102A] → RCA L/R → [AMP TPA3116D2 4.1] AMP zasilanie: +12V akumulator → bezpiecznik 20A → AMP VCC AMP FL+/FL- → głośnik przód lewy (fabryczny, złącze ISO) AMP FR+/FR- → głośnik przód prawy (fabryczny) AMP RL+/RL- → głośnik tył lewy (fabryczny) AMP RR+/RR- → głośnik tył prawy (fabryczny) AMP SUB+/SUB- → [kabel 2.5mm² 5m] → subwoofer 8" (bagażnik)

18.4 Kontroler obrotowy (Arduino Pro Micro USB HID)

D2 ← Enkoder CLK (+10kΩ pull-up) | D3 ← Enkoder DT (+10kΩ pull-up) D4 ← Enkoder SW (push) | D5 ← HOME | D6 ← BACK | D7 ← MEDIA D8 ← VOL+ | D9 ← VOL- | Wszystkie: active LOW, wewnętrzny pull-up USB micro-B → kabel 0.5m → OPi5+ USB hub

18.5 HC-SR04, DS18B20, Buzzer — bez zmian

HC-SR04: Shared TRIG (GPIO) → 4× TRIG | 4× ECHO → [1kΩ+2kΩ dzielnik] → GPIO DS18B20: GPIO → DQ + [4.7kΩ pull-up → 3.3V] | VDD→3.3V | GND→GND Buzzer: GPIO → [1kΩ] → BC547 → Buzzer → +5V | [1N4148 flyback]
